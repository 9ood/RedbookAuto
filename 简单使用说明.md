# 📱 小红书自动发布工具 - 超简单使用说明

> 这个工具可以帮你自动发布内容到小红书，就像有个小助手帮你点击发布按钮一样！

---

## 🎯 这个工具能做什么？

想象一下：
- 你写好了文章和准备好了图片
- 把它们放到一个文件夹里
- 工具会自动帮你发布到小红书
- 你甚至可以设置每天固定时间自动发布！

---

## 🚀 第一次使用（只需要做一次）

### 步骤 1：登录小红书
打开电脑上的"终端"程序（就像一个黑色的窗口），输入：
```
npx xhs-mcp login --timeout 120
```

然后：
1. 会弹出一个浏览器窗口
2. 用手机小红书扫码登录
3. 登录成功后，关闭浏览器就可以了

**这一步只需要做一次！** 以后就不用再登录了。

---

## 📝 准备你要发布的内容

### 步骤 1：找到"queue/pending"文件夹
在你的电脑上找到这个工具的文件夹，里面有个 `queue/pending` 文件夹。

### 步骤 2：创建一个新文件夹
在 `queue/pending` 里面，创建一个新文件夹，名字随便起，比如：
```
我的第一篇文章
```

### 步骤 3：放入你的图片
把你要发布的图片（比如 `图片1.jpg`、`图片2.jpg`）复制到这个文件夹里。

### 步骤 4：创建一个说明文件
在同一个文件夹里，创建一个叫 `meta.json` 的文件，写上：

```json
{
  "title": "这里写标题",
  "content": "这里写正文内容",
  "images": ["图片1.jpg", "图片2.jpg"]
}
```

**举个例子：**
```json
{
  "title": "今天的美食分享",
  "content": "今天做了超好吃的蛋糕！配方很简单，大家快来试试吧！",
  "images": ["蛋糕1.jpg", "蛋糕2.jpg"]
}
```

---

## 🎬 发布内容

### 方式 1：立即发布（内容会马上出现在小红书上）

打开"终端"，输入：
```
python3 publisher/run_once.py
```

然后：
1. 工具会自动打开浏览器
2. 自动上传图片
3. 自动填写标题和内容
4. 自动点击"发布"按钮
5. 完成！你的内容已经发布到小红书了！

### 方式 2：保存草稿（先不发布，以后手动发布）

如果你想先保存草稿，稍后自己手动发布，可以在 `meta.json` 里加上 `"draft": true`：

```json
{
  "title": "今天的美食分享",
  "content": "今天做了超好吃的蛋糕！",
  "images": ["蛋糕1.jpg", "蛋糕2.jpg"],
  "draft": true
}
```

然后运行：
```
python3 publisher/run_once.py
```

这样内容会保存为草稿，你可以稍后在小红书 App 里手动发布。

## ⏰ 设置自动定时发布（高级功能）

如果你想让工具每天固定时间自动发布（比如每天早上 8 点和下午 3 点），可以这样做：

### 步骤 1：安装定时任务
打开"终端"，输入：
```
./scripts/install_launchd.sh
```

### 步骤 2：完成！
现在工具会在每天早上 8:00 和下午 15:00 自动检查 `queue/pending` 文件夹，如果有内容就自动发布！

### 如果想取消定时发布
打开"终端"，输入：
```
./scripts/uninstall_launchd.sh
```

## ❓ 常见问题

### Q1: 我不会用"终端"怎么办？
**答：** "终端"就是电脑上的一个程序：
- Mac 电脑：按 `Command + 空格`，输入"终端"，回车
- 打开后会看到一个黑色或白色的窗口，可以输入命令

### Q2: 图片必须是 JPG 格式吗？
**答：** 不是！支持这些格式：
- JPG / JPEG
- PNG
- GIF
- WEBP

### Q3: 可以不写图片列表吗？
**答：** 可以！如果你不写 `"images"`，工具会自动找到文件夹里的所有图片。

### Q4: 发布失败了怎么办？
**答：** 
1. 检查是否已经登录（重新运行 `npx xhs-mcp login --timeout 120`）
2. 检查 `meta.json` 文件格式是否正确
3. 检查图片文件是否存在

### Q5: 怎么知道发布成功了？
**答：** 
1. 终端会显示"发布成功"的消息
2. 你可以在小红书 App 里查看你的主页
3. 查看 `logs/publisher.log` 文件，里面有详细记录

## 💡 小提示

1. **标题不要太长**：建议不超过 20 个字
2. **图片不要太多**：最多 18 张图片
3. **先测试**：第一次使用时，先发布一篇测试内容，确保一切正常
4. **保存草稿**：如果不确定内容，可以先保存草稿，稍后手动发布

---

## 📞 需要帮助？

如果遇到问题：
1. 查看 `logs/publisher.log` 文件，里面有详细的运行记录
2. 检查 `meta.json` 文件格式是否正确（可以用在线 JSON 检查工具）
3. 确保已经登录小红书
